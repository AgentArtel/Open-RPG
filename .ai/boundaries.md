# File-to-Agent Ownership Map

Every file in the repo maps to exactly one agent. When in doubt, check this file.

## Claude Code (Orchestrator)

### Coordination Files
- `AGENTS.md` — universal source of truth
- `CLAUDE.md` — orchestrator config
- `.ai/**` — task coordination, status, templates, workforce guide

### Documentation
- `docs/**` — architecture docs, ADRs, guides
- `idea/**` — project vision and research documents

### Root Configuration
- `package.json`
- `tsconfig.json`
- `tsconfig*.json`
- `rpg.toml`
- `.gitignore`
- `.env.example`

### Entry Points
- `src/server.ts`
- `src/client.ts`
- `src/modules/main/index.ts`

## Cursor (Implementation Specialist)

### Agent System
- `src/agents/core/**` — AgentRunner, LLM client abstraction, lane queue
- `src/agents/skills/**` — game command tool definitions (move, look, say, wait, emote)
- `src/agents/perception/**` — PerceptionEngine (game state → text)
- `src/agents/memory/**` — AgentMemory (conversation buffer, persistence)
- `src/agents/bridge/**` — GameChannelAdapter, RPGJS ↔ agent wiring

### Game Server
- `src/modules/main/server/events/**` — NPC/Event classes (@EventData)
- `src/modules/main/server/maps/**` — map classes + tmx files
- `src/modules/main/server/database/**` — items, weapons, skills
- `src/modules/main/server/player.ts` — player hooks

### Game Client
- `src/modules/main/client/characters/**` — spritesheet definitions
- `src/modules/main/client/gui/**` — GUI components
- `src/modules/main/client/sounds/**` — audio assets
- `src/modules/main/client/sprite.ts` — client sprite hooks

### Configuration
- `src/config/**` — agent personality YAML files

### Assets
- `maps/**` — Tiled .tmx map files
- `assets/sprites/**` — character spritesheets
- `assets/tilesets/**` — tileset images
